이미지 생성 모델로 만든 UI 목업을 Codex 같은 코딩 에이전트에게 바로 넘기면 결과가 자주 무너진다. 화면은 비슷해 보일 수 있지만 반응형이 깨지고, 컴포넌트가 일관되지 않고, 버튼 상태나 간격 규칙이 제각각이 되기 쉽다.

이 문제는 AI가 부족해서만 생기지 않는다. 중간 산출물이 빠졌기 때문에 생긴다. 사람 프론트엔드 개발자도 이미지 한 장만 보고 전체 앱을 안정적으로 구현하지 않는다. 먼저 디자인 시스템을 추출하고, 토큰과 컴포넌트를 정리한 뒤, 화면을 조립한다.

## 실패하는 방식

흔한 흐름은 이렇다. 먼저 “협업 앱 UI 목업을 만들어줘”라고 이미지 생성 모델에 요청한다. 그다음 결과 이미지를 보고 “이 디자인 그대로 웹앱으로 구현해줘”라고 코딩 에이전트에게 넘긴다.

이 방식이 실패하는 이유는 이미지 한 장에 구현 규칙이 명시되어 있지 않기 때문이다. 코딩 에이전트는 색상 토큰, 타이포그래피, 여백, 브레이크포인트, 컴포넌트 상태, 접근성 규칙, UI 프레임워크 선택까지 모두 추론해야 한다.

추론할 것이 너무 많으면 결과는 흔들린다. 어떤 값은 목업을 따라가고, 어떤 값은 프레임워크 기본값을 쓰고, 어떤 값은 에이전트가 임의로 만들어낸다.

## 올바른 순서

첫 단계는 목업 이미지를 만드는 것이다. 이때도 단순히 예쁜 화면보다 제품군, 사용자, 분위기, 정보 밀도를 함께 지정하는 편이 좋다.

두 번째 단계는 목업에서 디자인 시스템을 추출하는 것이다. 색상 팔레트, semantic color token, 타이포그래피 스케일, spacing, layout grid, core component, interaction state, 접근성 규칙을 분리한다.

세 번째 단계는 기존 UI 프레임워크와 맞추는 것이다. shadcn/ui, Radix UI, MUI, Chakra UI, Mantine 같은 선택지를 비교하고, 현재 디자인과 가장 가까운 기반을 고른다. 모든 컨트롤을 처음부터 만들 필요는 없다. 접근성과 키보드 인터랙션은 검증된 라이브러리의 도움을 받는 편이 낫다.

네 번째 단계는 디자인 시스템부터 구현하는 것이다. 색상, 폰트, 간격, radius, shadow 같은 토큰을 먼저 만들고, page shell, sidebar, top bar, content grid 같은 레이아웃을 잡는다. 그다음 Button, Input, Card, Modal, Tabs, Navigation 같은 기본 컴포넌트를 만든다.

마지막 단계에서 실제 화면을 조립한다. 이때 새 색상이나 새 간격을 마음대로 추가하지 않고, 앞에서 만든 토큰과 컴포넌트를 재사용해야 한다.

## 단계별 리뷰가 중요한 이유

AI에게 한 번에 모든 것을 맡기면 실패 원인을 찾기 어렵다. 레이아웃이 문제인지, 색상 토큰이 문제인지, 컴포넌트 상태가 문제인지, 반응형 규칙이 문제인지 섞여버린다.

따라서 작업은 작게 나누는 편이 좋다. 레이아웃 컨테이너만 구현하고 리뷰한다. 토큰만 구현하고 리뷰한다. 핵심 컴포넌트만 구현하고 리뷰한다. 마지막에 화면을 조립한다.

이 방식은 느려 보이지만 실제로는 빠르다. 잘못된 추론이 전체 앱으로 번지기 전에 멈출 수 있고, 각 단계가 커밋 단위로 남아 되돌리기도 쉽다.

## 프로젝트 메모리에 남길 규칙

UI 구현 프로젝트라면 AGENTS.md나 프로젝트 메모리에 다음 원칙을 남기는 것이 좋다.

- 디자인 시스템을 먼저 추출하고 구현한다.
- 데스크톱, 태블릿, 모바일 브레이크포인트를 명시한다.
- 버튼, 입력창, 카드, 모달 같은 컴포넌트는 재사용한다.
- hover, focus, disabled, loading, error 상태를 포함한다.
- 각 단계는 리뷰 가능한 작은 단위로 끝낸다.
- 최종 화면에서 새 토큰을 임의로 만들지 않는다.

## 정리

AI 목업 구현의 핵심은 이미지를 코드로 바로 바꾸는 것이 아니다. 이미지를 구현 가능한 규칙으로 번역하는 것이다.

좋은 워크플로우는 이미지 목업, 디자인 시스템, 토큰, 컴포넌트, 화면 조립으로 이어진다. 이 순서를 지키면 Codex 같은 에이전트는 훨씬 안정적으로 일한다. AI에게도 사람 개발자와 같은 업무 맥락이 필요하다.
