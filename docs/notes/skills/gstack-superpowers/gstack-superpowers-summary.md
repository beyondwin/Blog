# GStack + Superpowers 작업 방식 요약

이 문서는 [gstack-superpowers-analysis.md](gstack-superpowers-analysis.md)를 바탕으로, `GStack + Superpowers`를 실제 작업에 어떻게 쓰는지 빠르게 참고할 수 있도록 정리한 요약본이다.

## 1. 핵심 개념

이 조합의 핵심은 Claude에게 바로 구현을 시키는 것이 아니라, 아래 순서대로 작업을 구조화하는 데 있다.

`아이디어 정리 -> 문서화 -> 계획 수립 -> 작업 분리 -> 병렬 실행 -> 중간 리뷰`

즉, AI를 단순 생성기로 쓰는 방식이 아니라 `기획 파트너 + 설계자 + 병렬 작업 팀`처럼 운용하는 방식이다.

## 2. 각 도구의 역할

### GStack

- `Office Hours`
  - 작업 시작 전 아이디어를 정리하는 단계
  - 요구사항, 목적, 사용 상황, 문제점을 질문으로 구체화

- `Design Review`
  - 중간 결과물을 보면서 어색한 UI/디자인을 교정하는 단계
  - AI 특유의 과한 스타일이나 완성도 부족을 줄이는 역할

### Superpowers

- `Brainstorming`
  - 초안 문서나 아이디어에 빠진 구멍이 없는지 다시 점검
  - 방향성, UX/UI 아이디어, 누락된 요구사항 보강

- `Writing-Plans`
  - 구현 전에 단계별 실행 계획서를 작성
  - 작업 순서와 기준점을 고정

- `Git WorkTrees`
  - 기능별 작업 공간 분리
  - 여러 작업을 서로 섞이지 않게 병렬 진행

- `Subagent Driven Development`
  - 큰 작업을 작은 단위로 나눠 병렬 실행
  - 각 단위가 자기 컨텍스트에만 집중하게 함

- `자동 기록`
  - 의미 있는 작업 단위마다 기록을 남겨 흐름과 변경 이력을 관리

## 3. 표준 작업 순서

실제 작업은 아래 순서로 진행하면 된다.

1. `Office Hours`로 시작한다.
2. 대화 내용을 바탕으로 작업 문서를 정리한다.
3. `Brainstorming`으로 빠진 내용을 다시 점검한다.
4. `Writing-Plans`로 단계별 계획서를 만든다.
5. 계획서를 리뷰해서 한 번 더 검증한다.
6. `Git WorkTrees`로 작업 공간을 나눈다.
7. `Subagent Driven Development`로 구현을 병렬 진행한다.
8. 중간중간 `Design Review`로 결과물을 다듬는다.
9. 자동 기록을 활용해 변경 이력을 관리한다.

## 4. 단계별로 무엇을 얻는가

- 시작 단계:
  - 하고 싶은 일을 말로만 두지 않고 구조화할 수 있다.

- 문서화 단계:
  - “무엇을 왜 만드는지”가 명확해진다.

- 계획 단계:
  - 구현 순서와 기준점이 생겨 작업 도중 방향을 잃지 않는다.

- 작업 분리 단계:
  - 여러 작업이 서로 꼬이지 않는다.

- 병렬 실행 단계:
  - 큰 작업을 역할 단위로 나눠 품질과 속도를 함께 챙길 수 있다.

- 리뷰 단계:
  - 결과물의 어색함과 AI 슬롭을 줄일 수 있다.

## 5. 이 방식이 잘 맞는 작업

- 새 기능 기획부터 구현까지 한 번에 진행하는 작업
- 요구사항이 아직 흐릿한 프로젝트
- 화면, UX, 문서, 구현이 함께 얽힌 작업
- 동시에 여러 단위를 진행해야 하는 작업
- 방향이 자주 흔들리거나 결과물이 들쭉날쭉했던 작업

## 6. 이 방식의 장점

- 바로 구현하지 않아서 초반 오해를 줄인다.
- 계획서를 먼저 만들기 때문에 중간 이탈이 적다.
- 작업 공간을 나눠 컨텍스트 오염을 막는다.
- 병렬 처리로 큰 작업을 안정적으로 밀어붙일 수 있다.
- 리뷰 단계를 중간에 넣어 품질을 유지할 수 있다.

## 7. 실무용 한 줄 워크플로우

`Office Hours -> 문서화 -> Brainstorming -> Writing-Plans -> Plan Review -> WorkTrees -> Subagents -> Design Review`

## 8. 짧은 결론

`GStack + Superpowers` 조합은 Claude Code를 더 똑똑하게 만드는 도구 조합이라기보다, `작업 순서와 협업 구조를 만들어 주는 운영 방식`에 가깝다.

가장 중요한 포인트는 하나다.

`바로 만들지 말고, 먼저 정리하고 나눠서 진행한다.`
