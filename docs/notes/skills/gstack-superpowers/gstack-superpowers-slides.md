# GStack + Superpowers 발표자료용 초압축 버전

## 슬라이드 1. 제목

### GStack + Superpowers로 Claude Code 일 잘 시키는 방법

- 핵심은 툴 추가가 아니라 작업 순서 설계
- 바로 만들지 않고, 먼저 정리하고 나눠서 진행

---

## 슬라이드 2. 문제

### 왜 Claude Code 결과물이 자주 흔들릴까

- 요구사항이 흐릿한 상태에서 바로 구현 시작
- 큰 작업을 한 컨텍스트에서 한 번에 처리
- 중간 점검 없이 결과물만 보고 수정 반복

---

## 슬라이드 3. 핵심 아이디어

### 이 조합의 핵심

`아이디어 정리 -> 문서화 -> 계획 수립 -> 작업 분리 -> 병렬 실행 -> 중간 리뷰`

- AI를 생성기가 아니라 협업 구조로 운용
- 방향, 품질, 속도를 동시에 챙기는 방식

---

## 슬라이드 4. GStack 역할

### GStack는 시작과 점검 담당

- `Office Hours`
  - 시작 전에 요구사항 정리
  - 왜 만드는지, 누가 쓰는지, 무엇이 불편한지 명확화

- `Design Review`
  - 중간 결과물 디자인/완성도 점검
  - AI 슬롭, 어색한 UI, 과한 스타일 보정

---

## 슬라이드 5. Superpowers 역할

### Superpowers는 실행 구조 담당

- `Brainstorming`: 빠진 요구사항과 아이디어 구멍 점검
- `Writing-Plans`: 구현 전 단계별 계획서 작성
- `Git WorkTrees`: 기능별 작업 공간 분리
- `Subagent Driven Development`: 큰 작업 병렬 실행
- `자동 기록`: 작업 흐름과 변경 이력 관리

---

## 슬라이드 6. 실제 작업 순서

### 실무용 표준 플로우

1. `Office Hours`
2. 문서화
3. `Brainstorming`
4. `Writing-Plans`
5. Plan Review
6. `WorkTrees`
7. `Subagents`
8. `Design Review`

---

## 슬라이드 7. 이 방식의 장점

### 무엇이 달라지나

- 시작 전에 생각이 정리됨
- 구현 전에 기준점이 생김
- 작업이 서로 꼬이지 않음
- 큰 작업을 병렬로 안정적으로 진행 가능
- 결과물을 중간에 다듬어 품질 유지 가능

---

## 슬라이드 8. 잘 맞는 작업

### 이런 경우 특히 효과적

- 새 기능 기획부터 구현까지 한 번에 진행할 때
- 요구사항이 아직 흐릿할 때
- UI, UX, 문서, 구현이 함께 얽혀 있을 때
- 여러 작업을 동시에 진행해야 할 때

---

## 슬라이드 9. 결론

### 한 줄 결론

`GStack + Superpowers`는 Claude Code를 더 똑똑하게 만드는 조합이 아니라,
**작업을 잘 시작하고, 잘 나누고, 잘 끝내게 만드는 조합**이다.

### 기억할 문장

`바로 만들지 말고, 먼저 정리하고, 계획을 세우고, 나눠서 진행한다.`
